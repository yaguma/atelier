# RPGテンプレート

JRPG、アクションRPG、ターン制RPGなど、ロールプレイングゲーム向け。

## 典型的なプレイサイクル

### 戦闘ループ
```
① 状況提示: 敵の状態、味方の状態、行動順
② プレイヤーの選択: 攻撃/スキル/アイテム/防御/逃走
③ 結果反映: ダメージ計算、状態異常、戦闘終了判定
④ フィードバック: ダメージ表示、エフェクト、経験値獲得
→ ①に戻る or 戦闘終了
```

### 探索ループ
```
① 状況提示: マップ、NPC、イベントポイント
② プレイヤーの選択: 移動、会話、調べる
③ 結果反映: ストーリー進行、アイテム獲得
④ フィードバック: 会話テキスト、イベントシーン
```

## ジャンル特有の要素

### 成長システム
- レベルアップ方式
- ステータス成長
- スキル習得

### パーティ
- パーティ人数
- 編成の自由度
- 役割分担（タンク/ヒーラー/アタッカー）

### 装備
- 装備スロット
- 強化システム
- レアリティ

### ストーリー
- メインストーリー構成
- サブクエスト
- 分岐/マルチエンディング

## ヒアリング質問

## 出力テンプレート

```markdown
# {ゲーム名} 要件定義書（RPG）

## 1. プレイサイクル

### 戦闘ループ
① 状況提示: ターン順表示、敵味方ステータス
② プレイヤーの選択: コマンド選択（攻撃/スキル/アイテム/防御）
③ 結果反映: ダメージ/回復/状態変化
④ フィードバック: アニメーション、数値表示

### 探索ループ
① 状況提示: マップ、イベントアイコン
② プレイヤーの選択: 移動、NPC会話
③ 結果反映: ストーリー進行、情報獲得
④ フィードバック: テキスト表示、カットシーン

## 2. 勝敗条件
- 勝利条件: ラスボスを撃破 / ストーリー完結
- 敗北条件: パーティ全滅
- 継続条件: コンティニュー可能 / 最終セーブから再開

## 3. プレイヤー操作
- 攻撃する
- スキルを使う
- アイテムを使う
- 防御する
- 逃げる
- 装備を変更する
- パーティを編成する

## 4. ゲーム要素

### キャラクター
- 仲間キャラ数: {N}人
- パーティ最大人数: {N}人
- ステータス: HP/MP/攻撃/防御/素早さ/魔力
- クラス/職業: {クラスリスト}

### スキル
- 習得方法: レベルアップ/アイテム/イベント
- 種類: 攻撃/回復/補助/状態異常
- コスト: MP/TP

### 装備
- スロット: 武器/防具/アクセサリ
- レアリティ: {レアリティ}
- 強化: {強化システム}

### 敵
- 種類: 雑魚/ボス/隠しボス
- ドロップ: アイテム/お金/経験値
- 属性/弱点: {属性システム}

### マップ
- ワールド構成: {エリアリスト}
- ダンジョン数: {N}
- 町/拠点数: {N}

## 5. 数値設計

| 変数 | 範囲 | 初期値 |
|-----|------|-------|
| 最大レベル | 50〜100 | 99 |
| パーティ人数 | 3〜6 | 4 |
| 初期HP | 100〜500 | 200 |
| 初期MP | 20〜100 | 50 |
| プレイ時間(想定) | 20〜60時間 | 30時間 |

## 6. 想定プレイフロー

### 序盤（レベル1〜10）
- 状況: 主人公1人、基本スキル習得
- 行動: 最初のダンジョン攻略、仲間1人加入
- 結果: 基本戦術習得、ストーリー導入完了

### 中盤（レベル30〜50）
- 状況: パーティ4人、上位スキル習得
- 行動: 複数ダンジョン攻略、サブクエスト消化
- 結果: ストーリー核心、パーティビルド確立

### 終盤（レベル70〜99）
- 状況: 最終装備、全スキル習得
- 行動: ラストダンジョン攻略、ラスボス戦
- 結果: エンディング到達
```
