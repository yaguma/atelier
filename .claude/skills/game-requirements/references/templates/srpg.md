# SRPGテンプレート

ファイアーエムブレム、タクティクスオウガ、FFタクティクスなどのシミュレーションRPG向け。

## 典型的なプレイサイクル

```
① 状況提示: マップ、敵配置、味方ユニット状態、勝利条件
② プレイヤーの選択: ユニット移動→行動（攻撃/スキル/待機）
③ 結果反映: 戦闘計算、HP変化、状態異常、撃破判定
④ フィードバック: 戦闘アニメ、ダメージ表示、撃破演出
→ 全味方行動終了 → 敵ターン → ①に戻る
```

## ジャンル特有の要素

### ユニット
- クラス/職業
- 成長タイプ
- 個性/固有スキル
- 装備スロット

### マップ
- グリッドタイプ（正方形/ヘックス）
- 地形効果
- 高低差

### 戦闘計算
- 命中/回避
- クリティカル
- 属性相性
- 武器相性（三すくみなど）

### パーマデス
- ユニットロストの有無
- 復活手段
- カジュアルモード

## ヒアリング質問

### ユニットについて
- 「ユニットは何人くらい？」
- 「クラス/職業はある？」
- 「成長はランダム？固定？」

### マップについて
- 「マス目は正方形？ヘックス？」
- 「地形効果はある？」
- 「高低差はある？」

### 戦闘について
- 「三すくみのような相性はある？」
- 「パーマデス（ユニットロスト）はある？」

## 出力テンプレート

```markdown
# {ゲーム名} 要件定義書（SRPG）

## 1. プレイサイクル

### 味方フェイズ
① 状況提示: マップ全体、敵味方位置、行動可能ユニット
② プレイヤーの選択: ユニット選択→移動→行動
③ 結果反映: 戦闘計算、HP変化、撃破判定
④ フィードバック: 戦闘アニメ、経験値獲得

### 敵フェイズ
- 敵AIによる移動と攻撃
- プレイヤーは反撃可能

## 2. 勝敗条件
- 勝利条件: 敵将撃破 / 敵全滅 / 拠点制圧 / 生存
- 敗北条件: 主人公撃破 / 味方全滅 / 防衛失敗
- パーマデス: {あり/なし/選択可能}

## 3. プレイヤー操作
- ユニットを選択する
- 移動先を決める
- 攻撃対象を選ぶ
- スキルを使う
- アイテムを使う
- 待機する
- ターンを終了する

## 4. ゲーム要素

### ユニット
- 出撃可能数: {N}人
- ステータス: HP/力/技/速さ/守備/魔防/移動
- クラス: {クラスリスト}

### クラス
| クラス | 移動 | 武器 | 特性 |
|-------|------|------|------|
| 歩兵 | 5 | 剣 | バランス型 |
| 騎兵 | 8 | 槍 | 高機動 |
| 重装 | 4 | 斧 | 高守備 |
| 弓兵 | 5 | 弓 | 間接攻撃 |
| 魔道士 | 5 | 魔法 | 魔法攻撃 |

### 武器相性
```
剣 > 斧 > 槍 > 剣（三すくみ）
弓 > 飛行（特効）
魔法 > 重装（特効）
```

### 地形
| 地形 | 回避補正 | 防御補正 | 移動コスト |
|------|---------|---------|-----------|
| 平地 | 0 | 0 | 1 |
| 森 | +20 | +1 | 2 |
| 砦 | +30 | +2 | 2 |

### 戦闘計算
- 命中率 = 技×2 + 武器命中 - 敵回避
- ダメージ = 力 + 武器威力 - 敵守備
- 追撃条件 = 速さ差{N}以上
- クリティカル = 技/2

### マップ
- グリッド: {N}x{M}
- ステージ数: {N}
- 高低差: {あり/なし}
```
