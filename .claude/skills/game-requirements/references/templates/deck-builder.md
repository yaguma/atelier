# デッキ構築テンプレート

Slay the Spire、Monster Train、Inscryptionなどのデッキ構築ゲーム向け。

## 典型的なプレイサイクル

```
① 状況提示: 手札が配られる、敵の行動予告が表示される
② プレイヤーの選択: カードを選んで使う（複数枚可能な場合も）
③ 結果反映: 敵HP減少、バフ/デバフ付与、カード効果発動
④ フィードバック: ダメージ数値、エフェクト、効果音
→ 敵ターン → ①に戻る
```

## ジャンル特有の要素

### デッキ
- 初期デッキ枚数
- 最大デッキ枚数
- カードの追加/削除タイミング

### カード
- レアリティ（コモン/アンコモン/レア/レジェンダリー）
- コスト（マナ/エナジー）
- 効果タイプ（攻撃/防御/スキル/パワー）

### 戦闘
- ターン制 or リアルタイム
- 複数敵との戦闘
- ターン開始時のドロー枚数

### メタ進行
- ラン間の永続アンロック
- デッキの改造タイミング

## 出力テンプレート

```markdown
# {ゲーム名} 要件定義書（デッキ構築）

## 1. プレイサイクル
① 状況提示: 手札{N}枚配布、敵の行動予告表示
② プレイヤーの選択: カードを選択して使用
③ 結果反映: カード効果適用、敵/プレイヤー状態変化
④ フィードバック: ダメージ表示、エフェクト再生

## 2. 勝敗条件
- 勝利条件: 全ての敵を倒す / 最終ボスを撃破
- 敗北条件: プレイヤーHPが0になる
- 継続不能条件: なし（デッキは尽きない想定）

## 3. プレイヤー操作
- カードを使う
- カードを捨てる（任意）
- ターンを終了する
- デッキを確認する
- 捨て札を確認する

## 4. ゲーム要素

### デッキ
- 初期枚数: {N}枚
- 最大枚数: {N}枚
- 構成: 攻撃{X}枚、防御{Y}枚、スキル{Z}枚

### カード
- レアリティ: コモン/アンコモン/レア
- コスト: 0〜{N}
- 効果: 攻撃/防御/バフ/デバフ/ドロー

### プレイヤー
- HP: {N}
- エナジー: {N}/ターン
- ブロック（毎ターンリセット）

### 敵
- HP: {N}〜{M}
- 行動パターン: 攻撃/防御/バフ/召喚

## 5. 数値設計

| 変数 | 範囲 | 初期値 |
|-----|------|-------|
| プレイヤーHP | 50〜100 | 70 |
| エナジー/ターン | 3〜5 | 3 |
| 手札枚数 | 5〜7 | 5 |
| 初期デッキ枚数 | 10〜15 | 10 |
| カード追加/戦闘 | 1〜3 | 1 |

## 6. 想定プレイフロー

### 1ターン目
- 状況: 手札5枚、エナジー3、敵HP30（攻撃8予告）
- 行動: 攻撃カード(コスト1)×2、防御カード(コスト1)×1
- 結果: 敵HP20、ブロック5獲得 → 敵攻撃で3ダメージ

### 2ターン目
- 状況: 手札5枚、エナジー3、敵HP20（強攻撃15予告）
- 行動: 防御カード×2、バフカード
- 結果: ブロック15獲得 → ダメージ0
```
