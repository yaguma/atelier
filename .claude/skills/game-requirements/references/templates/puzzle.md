# パズルテンプレート

マッチ3、落ち物パズル、謎解きパズルなど、パズルゲーム向け。

## 典型的なプレイサイクル

### マッチ系パズル
```
① 状況提示: 盤面状態、目標、残り手数/時間
② プレイヤーの選択: ピースの移動/入れ替え/配置
③ 結果反映: マッチ判定、消去、落下、連鎖
④ フィードバック: 消去エフェクト、スコア加算、連鎖演出
→ ①に戻る
```

### 謎解きパズル
```
① 状況提示: パズル状態、ヒント、目標
② プレイヤーの選択: オブジェクト操作、アイテム使用
③ 結果反映: パズル状態変化、正解/不正解判定
④ フィードバック: 状態変化演出、正解演出
→ ①に戻る or クリア
```

## ジャンル特有の要素

### 盤面
- グリッドサイズ
- ピースの種類
- 特殊マス/ギミック

### 目標
- スコア達成
- 特定ピース消去
- 制限ターン/時間内クリア

### 連鎖/コンボ
- 連鎖システム
- コンボボーナス
- 特殊ピース生成

### 難易度曲線
- ステージ構成
- ギミック解放タイミング

## ヒアリング質問

### 基本ルールについて
- 「どうやってピースを動かす？」
- 「何が揃ったら消える？」
- 「目標は何？（スコア？ターン？特定消去？）」

### ギミックについて
- 「特殊なピースやマスはある？」
- 「連鎖システムはある？」

### 進行について
- 「ステージ制？エンドレス？」

## 出力テンプレート

```markdown
# {ゲーム名} 要件定義書（パズル）

## 1. プレイサイクル
① 状況提示: 盤面{N}x{M}、残り手数{X}
② プレイヤーの選択: ピースをスワイプ/タップで操作
③ 結果反映: マッチ消去、落下処理、連鎖判定
④ フィードバック: 消去エフェクト、スコア表示、連鎖演出

## 2. 勝敗条件
- 勝利条件: 目標達成（スコア/消去数/特殊条件）
- 敗北条件: 手数/時間切れで目標未達成
- 詰み条件: 有効な手がない（リシャッフル発動）

## 3. プレイヤー操作
- ピースを選ぶ
- ピースを動かす
- 特殊スキルを発動する
- ヒントを使う
- リトライする

## 4. ゲーム要素

### 盤面
- サイズ: {N}x{M}
- ピース種類: {N}種類
- 特殊マス: {リスト}

### ピース
- 通常ピース: {色/形リスト}
- 特殊ピース: {生成条件と効果}

### 目標タイプ
- スコア達成: {N}点以上
- ピース消去: {種類}を{N}個
- ターン制限: {N}手以内
- 時間制限: {N}秒以内

### ギミック
- {ギミック名}: {効果}

## 5. 数値設計

| 変数 | 範囲 | 初期値 |
|-----|------|-------|
| 盤面サイズ | 5x5〜9x9 | 7x7 |
| ピース種類 | 4〜7 | 5 |
| 基本手数 | 10〜30 | 20 |
| マッチ必要数 | 3〜5 | 3 |
| 連鎖ボーナス | 1.5〜3倍 | 2倍 |

## 6. 想定プレイフロー

### ステージ1（チュートリアル）
- 状況: 5x5盤面、3種ピース、手数20
- 行動: 3マッチを作る基本操作
- 結果: 基本ルール習得

### ステージ10（ギミック導入）
- 状況: 7x7盤面、5種ピース、障害物あり
- 行動: 障害物を避けながらマッチ
- 結果: ギミック対応習得

### ステージ50（高難度）
- 状況: 7x7盤面、複合ギミック、手数15
- 行動: 連鎖を活用した効率プレイ
- 結果: 高度テクニック必要
```
