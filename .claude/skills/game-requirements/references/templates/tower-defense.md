# タワーディフェンステンプレート

Kingdom Rush、Bloons TD、アークナイツなどのタワーディフェンス向け。

## 典型的なプレイサイクル

```
① 状況提示: Wave開始、敵出現、ルート表示、リソース状況
② プレイヤーの選択: タワー配置/アップグレード/スキル発動
③ 結果反映: タワー攻撃、敵撃破、敵進行、ライフ減少
④ フィードバック: 攻撃エフェクト、撃破演出、収入表示
→ Wave終了 → 次Wave or ステージクリア
```

## ジャンル特有の要素

### タワー
- タワー種類（攻撃/減速/支援）
- 配置ルール（自由配置/固定位置）
- アップグレード経路
- 相性/シナジー

### 敵（Wave）
- Wave構成
- 敵の種類と特性
- ルート（固定/複数）

### 経済
- 初期リソース
- 敵撃破時の収入
- Wave間ボーナス

### 拠点
- ライフシステム
- 拠点スキル/能力

## ヒアリング質問

### タワーについて
- 「タワーは何種類？」
- 「配置は自由？それとも固定位置？」
- 「アップグレードはある？分岐は？」

### 敵について
- 「敵のルートは1本？複数？」
- 「特殊な敵（飛行、高速、高HP）はいる？」

### 経済について
- 「お金はどう稼ぐ？」
- 「タワーの売却は可能？」

## 出力テンプレート

```markdown
# {ゲーム名} 要件定義書（タワーディフェンス）

### Wave中
① 状況提示: 敵出現、進行状況、リソース
② プレイヤーの選択: タワー配置/アップグレード/スキル
③ 結果反映: タワー攻撃、敵撃破/通過
④ フィードバック: 攻撃エフェクト、撃破報酬

### Wave間
- リソース確認
- タワー配置/アップグレード準備
- 次Wave情報確認

- 勝利条件: 全Waveを凌ぎ切る

## 3. プレイヤー操作
- タワーを配置する
- タワーをアップグレードする
- タワーを売却する
- スキルを発動する
- ゲーム速度を変更する
- 次Waveを早める

## 4. ゲーム要素

### マップ
- 配置可能箇所: {N}箇所

## 6. 想定プレイフロー

### Wave 1-5（序盤）
- 状況: 資金300、雑魚敵のみ

- 結果: 基本戦術習得、資金500程度

### Wave 10-15（中盤）
- 状況: 資金1000、特殊敵出現
- 行動: タワーアップグレード、対空配置

### Wave 20-25（終盤）
```
