# 選択肢設計ガイド

## 選択肢の定義フォーマット

各選択肢は以下の4要素で定義する：

```
名前：
即時効果：
将来への影響：
リスク：
```

## 選択肢の種類（3〜5個を組み合わせる）

| 種類 | 特徴 | 例 |
|------|------|-----|
| 安全 | 小リターン、低リスク | 防御、回復、待機 |
| 危険 | 大リターン、高リスク | 強攻撃、賭け |
| 準備 | 即時効果なし、未来有利 | 充電、蓄積、観察 |
| 消費 | リソース消費で強力効果 | 必殺技、アイテム使用 |
| 条件 | 状況依存で効果変動 | カウンター、連携 |

## 設計例

### 強攻撃
- 即時：敵HP -5
- 将来：次のターン手札 -1
- リスク：失敗率20%

### 防御
- 即時：次の被ダメージ -3
- 将来：なし
- リスク：なし

### チャージ
- 即時：なし
- 将来：次の攻撃ダメージ 2倍
- リスク：このターン無防備

## ランダム要素の入れ方

### OK
- 成功/失敗の確率
- ダメージ±1〜2の揺れ
- 複数結果からの抽選

### NG
- 行動内容自体が変わる
- 突然の即死
- 選択の意味がなくなる揺れ

## チェックポイント

- [ ] 「将来への影響」がある選択肢が1つ以上あるか
- [ ] 常に正解な選択肢がないか
- [ ] 常にハズレな選択肢がないか
