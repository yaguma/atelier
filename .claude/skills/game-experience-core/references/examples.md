## カードゲーム

### 中核判断
```
プレイヤーは常に「このカードを今使うべきか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 強いカードを今使う
結果: 敵は倒せるが手札が枯れる
学習: 次は温存タイミングを考えよう
```

### リソース設計
- 手札枚数（有限、補充に条件あり）
- マナ/コスト（ターン毎にリセット or 蓄積）
- デッキ枚数（使い切ると不利）

### 失敗設計
- 手札を使い切って窮地に → 次はリソース管理を意識
- キーカードを早く使いすぎた → デッキ構築を見直す

### 面白さの言語化
```
このゲームは
「限られた手札とマナの中で、
　最適なカードプレイを選び続けることが楽しい」
```

## ローグライク

### 中核判断
```
プレイヤーは常に「リスクを取るべきか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 危険だが報酬の高い部屋に入る
結果: レアアイテムを獲得したが体力が半分に
学習: このビルドなら攻めても良さそう
```

### リソース設計
- HP（永続的、回復が限定的）
- アイテムスロット（持てる量に制限）
- ゴールド（ショップでの選択を迫る）

### 失敗設計
- 死亡時に「あそこで回復を取るべきだった」と分かる
- ビルドの方向性ミスを次回に活かせる

### 面白さの言語化
```
このゲームは
「永続的なリソース消耗の中で、
　リスクと報酬を天秤にかけ続けることが楽しい」
```

## ストラテジー

### 中核判断
```
プレイヤーは常に「何に投資すべきか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 防衛より経済拡大を優先
結果: 序盤ラッシュに対応できず敗北
学習: 相手のスタイルを偵察してから判断しよう
```

### リソース設計
- 資源（複数種類、生産と消費のバランス）
- 時間（ビルド順序が勝敗を分ける）
- 情報（偵察で得る敵の動向）

### 失敗設計
- リプレイで敗因を分析できる
- 別戦略のカウンターを学習

### 面白さの言語化
```
このゲームは
「不完全な情報と有限の資源の中で、
　長期的な投資判断を選び続けることが楽しい」
```

## アクション

### 中核判断
```
プレイヤーは常に「攻めるか守るか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 敵の攻撃後に反撃を狙う
結果: タイミングが合って大ダメージ
学習: この敵のパターンを覚えた
```

### リソース設計
- HP（ミスの蓄積）
- スタミナ（回避・攻撃のリソース）
- 時間（反応速度が問われる）

### 失敗設計
- 被弾時に「今の攻撃は避けられたはず」と分かる
- パターンを覚えることで上達を実感

### 面白さの言語化
```
このゲームは
「限られた反応時間の中で、
　攻守のタイミングを見極め続けることが楽しい」
```

## パズル

### 中核判断
```
プレイヤーは常に「この手順で合っているか？」を考えている
```

### 判断→結果→学習ループ
```
判断: この順番でブロックを動かす
結果: 行き詰まりが発生
学習: 先にこちらを処理すべきだった
```

### リソース設計
- 手数（限られた操作回数）
- 空間（限られた配置場所）
- 時間（制限時間がある場合）

### 失敗設計
- やり直しが容易
- 失敗パターンから正解へのヒントが得られる

### 面白さの言語化
```
このゲームは
「限られた手数と空間の中で、
　最適な手順を発見し続けることが楽しい」
```

## 経営シミュレーション

### 中核判断
```
プレイヤーは常に「今何を優先すべきか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 新店舗を出店する
結果: 資金繰りが悪化するが将来の収益基盤ができる
学習: キャッシュフロー管理が重要
```

### リソース設計
- 資金（成長とリスクのバランス）
- 時間（市場の変化、機会損失）
- 人材（配置と育成）

### 失敗設計
- 倒産しても原因が分析できる
- 別のアプローチで再挑戦可能

### 面白さの言語化
```
このゲームは
「限られた資金と変化する市場の中で、
　成長戦略を選び続けることが楽しい」
```

## RPG

### 中核判断
```
プレイヤーは常に「どのキャラ/スキルを育てるべきか？」を考えている
```

### 判断→結果→学習ループ
```
判断: 火属性魔法を集中的に強化
結果: 火耐性の敵に苦戦
学習: バランス型のパーティ構成も検討しよう
```

### リソース設計
- 経験値/レベル（成長リソース）
- 装備/スキルスロット（選択を強制）
- 消耗品（ボス戦での切り札）

### 失敗設計
- 全滅時に戦略の見直しポイントが分かる
- レベリングやビルド変更でリカバリー可能

### 面白さの言語化
```
このゲームは
「限られた成長リソースと装備枠の中で、
　最強のパーティを構築し続けることが楽しい」
```

## 分析時の注意点

既存ゲームを分析する際は以下に注意:

1. **表面的な判断に惑わされない** - 「敵を倒す」は判断ではなく目標
2. **最も頻繁な判断を見つける** - レアケースではなく常に考えていること
3. **ジャンルの枠を超えて考える** - ローグライクでもパズル的判断がメインの場合もある
