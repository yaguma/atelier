## 概要

### 背景

旧ディレクトリ削除後、プロジェクト全体のインポートパスが正しく機能していることを最終確認する必要がある。

### 目的

プロジェクト全体のインポートパスを最終確認し、コンパイルエラーやリントエラーがないことを確認する。

### 現状

- 旧ディレクトリが削除済み
- 新しいfeatures構造に完全移行済み

## 実装計画

### ステップ1: TypeScriptコンパイル確認（1時間）

1. `pnpm tsc --noEmit`を実行
2. コンパイルエラーがあれば修正
3. 型エラーの解消

### ステップ2: リント確認（1時間）

1. `pnpm lint`を実行
2. エラーがあれば修正
3. 警告も可能な限り解消

### ステップ3: 未使用コード削除（1時間）

1. 未使用インポートの検出
2. 未使用インポートの削除
3. 未使用エクスポートの確認

### ステップ4: 最終確認（1時間）

1. 開発サーバー起動確認
2. ビルド成功確認
3. 全テスト実行
