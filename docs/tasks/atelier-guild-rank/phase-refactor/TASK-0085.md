## 概要

### 背景

FCIS（Functional Core, Imperative Shell）原則に基づき、インベントリ関連のビジネスロジックを純粋関数として再実装する必要がある。純粋関数化することで、テスタビリティと保守性が向上する。

### 目的

InventoryService、MaterialServiceを純粋関数として再実装し、addMaterial、removeMaterial等の操作を副作用のない関数として提供する。

### 現状

- 既存のサービスがクラスベースで実装されている可能性
- 状態変更が直接行われている可能性
