## 概要

### 背景

リファクタリングの一環として、デッキ機能に関連する型定義を`features/deck/types/`に集約する必要がある。現在、型定義がdomain層に散在しており、機能単位での管理が困難になっている。

### 目的

デッキ機能の型定義を`src/features/deck/types/`に移動し、型の一元管理を実現する。

### 現状

- Card型: `src/domain/entities/Card.ts`
- IDeckService: `src/domain/services/DeckService.ts`内に定義
