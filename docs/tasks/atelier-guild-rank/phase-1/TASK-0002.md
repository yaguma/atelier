## 2. 実装内容

### 2.1 Clean Architectureディレクトリ構造 🔵

```
src/
├── domain/                    # ドメイン層
│   ├── entities/              # エンティティ
│   ├── value-objects/         # 値オブジェクト
│   ├── services/              # ドメインサービス
│   └── interfaces/            # リポジトリインターフェース
├── application/               # アプリケーション層
│   ├── services/              # アプリケーションサービス
│   ├── use-cases/             # ユースケース
│   └── events/                # イベントバス
├── infrastructure/            # インフラストラクチャ層
│   ├── repositories/          # リポジトリ実装
│   ├── storage/               # ストレージ実装
│   └── loaders/               # データローダー
├── presentation/              # プレゼンテーション層
│   ├── scenes/                # Phaserシーン
│   ├── ui/                    # UIコンポーネント
│   └── managers/              # UI管理
├── shared/                    # 共有コード
│   ├── types/                 # 型定義
│   ├── constants/             # 定数
│   └── utils/                 # ユーティリティ
└── main.ts                    # エントリーポイント
```

### 2.2 各層のindex.ts作成 🟡

各ディレクトリにindex.tsを作成し、エクスポートを集約する。

### 2.3 アセットディレクトリ 🟡

```
public/
├── assets/
│   ├── images/
│   ├── audio/
│   └── data/
│       └── master/            # マスターデータJSON
```

### 2.4 テストディレクトリ 🟡

```
tests/
├── unit/
│   ├── domain/
│   ├── application/
│   └── infrastructure/
├── integration/
└── e2e/
```
