### 1. `Math.random` のグローバル上書きが復元されない

- **Perspective**: Security, Error Handling, QA Leader, Performance（4名が指摘）
- **Severity**: Warning (W2/W3: 副作用・テスト品質)
- **File**: `atelier-guild-rank/src/shared/utils/debug.ts:247-256`
- **Issue**: 新規追加の `setRandomSeed()` が `Math.random` をグローバルに永続的に置換するが、元の `Math.random` を保存・復元する仕組みがない。E2Eテストの `beforeEach` で毎回実行されるが、テスト後に復元されないためテスト間の独立性が損なわれる可能性がある。
- **Suggested Fix**: オリジナルの `Math.random` を保存し、復元関数を返すパターンに変更する。
  ```typescript
  static setRandomSeed(seed: number): () => void {
    const originalRandom = Math.random;
    let state = seed;
    Math.random = () => {
      state += 0x6d2b79f5;
      let z = state;
      z = Math.imul(z ^ (z >>> 15), z | 1);
      z ^= z + Math.imul(z ^ (z >>> 7), z | 61);
      return ((z ^ (z >>> 14)) >>> 0) / 4294967296;
    };
    return () => { Math.random = originalRandom; };
  }
  ```
  テスト側では `afterEach` で復元関数を呼ぶか、`restoreRandomSeed()` メソッドを別途追加する。
