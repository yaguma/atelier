# アトリエ錬金術ゲーム - プロトタイプ設計概要

## 目的

コンセプトデザインで定義した「面白さの仮説」を**最短で壊しに行く**プロトタイプを作成する。

## 面白さの仮説

> 「在庫（カード）が減り続ける恐怖の中で、今使うか温存するかを選び続けることが楽しい」

### 中核の判断

> 「このカードを今使うべきか？」

- 判断場面: 毎ターン、手札プレイ時
- 選択肢: 今使って依頼達成 / 温存 / 捨てて別カードを引く
- 面白さ: **使ったカードは二度と戻らない（在庫が減る恐怖）**

---

## 検証項目（3つ）

プレイしながら自問する：

1. **判断に迷いが生まれるか**
   - どの選択肢を押すか一瞬迷ったか？
   - 毎回同じボタンを押していないか？

2. **選択→結果→学習が1分以内に回るか**
   - ターン所要時間: 約25秒/ターン想定
   - 1プレイ時間: 3〜5分で1サイクル完結

3. **もう1回試したくなるか**
   - 「あのカードを温存すれば...」と思うか
   - 戦略の多様性が見えるか

---

## プロトタイプの制約

### 画面に存在してよいもの

- 状態表示（デッキ残量 / エネルギー / 安定度）
- 選択肢（カードボタン）
- 結果表示（数値変化・ログ）
- リセットボタン

### 存在してはいけないもの

- マップ遷移
- 演出アニメーション
- 複数画面
- セーブ / ロード

---

## 完成基準

- [ ] 画面は1枚のみ
- [ ] 3〜5分で1サイクル完了
- [ ] ルール説明なしでも触れる
- [ ] 5回くらい連続で試したくなる
- [ ] 「このカードを今使うべきか？」の判断が発生する
